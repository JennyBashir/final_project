# TODO Planner 

## Описание проекта 
Это простой веб-сервер, который выполняет функции планировщика задач (TODO-лист).

## С его помощью можно:
 - Добавлять задачи с датой, названием и комментарием.
 - Указывать сроки повторения задачи.
 - Смотреть список ближайших задач
 - Отмечать выполненные задач (при повторяющейся задаче - дата автоматически переносится).
 - Редактировать и удалять добавленные задачи.

Все задачи хранятся в базе данных SQLite.

В проекте реализованы все задания со звездочкой, кроме аутентификации. 

## Реализованные задачи:
- Настройка порта сервера через переменную окружения `TODO_PORT`
- Возможность указать путь к файлу базы данных черех `TODO_DBFILE`
- Поддержка правил повторения задач по неделям и месяцам (`w` и `m`)
- Поиск задач по названию, комментарию или конкретной дате.
- Создание Dockerfile для запуска сервера в контейнере.
- Оформление README.md с описанием проекта.

## Как запустить сервер на компьютере

### 1. Скачайте проект 
git clone git@github.com:JennyBashir/final_project.git
cd final_project
### 2. Установите зависимости
go mod tidy
### 3. Запустите сервер
go run main.go
### 4. Откройте браузер и перейдите по адресу: http://localhost:7540
По умолчанию сервер слушает порт 7540.
### 5. Если хотите другой порт:
export TODO_PORT=8080
go run main.go
### 6. Если хотите другой путь к базе данных:
export TODO_DBFILE=/<путь>/scheduler.db
go run main.go

## Тестирование

### Запуск всех тестов:
go test ./tests

### Тесты по отдельности:
go test -run ^TestApp$ ./tests       # проверка запуска сервера
go test -run ^TestDB$ ./tests        # проверка базы данных
go test -run ^TestAddTask$ ./tests   # добавление задачи
go test -run ^TestTasks$ ./tests     # получение списка задач
go test -run ^TestTask$ ./tests      # редактирование задачи
go test -run ^TestDone$ ./tests      # выполнение задачи
go test -run ^TestDelTask$ ./tests   # удаление задачи

## Как запустить через Docker

### 1. Сборка образа
docker build -t todo-server:latest .

### 2. Запуск контейнера:
#### Вариант 1  
(базовый запуск с существующей в проекте базой данных):

docker run -it --rm \
  -p 7540:7540
  -v $(pwd)/scheduler.db:/data/scheduler.db \
  todo-server:latest

#### Вариант 2 
(запуск с настройкой переменных окружения):

docker run -it --rm \
  -p 8080:8080 \
  -v $(pwd)/scheduler.db:/data/scheduler.db \
  -e TODO_PORT=8080 \
  -e TODO_DBFILE=/data/scheduler.db \
  todo-server:latest

Этот способ позволяет собственный порт сервера, а также использовать вашу базу данных или любую другую,
которую вы передадите в -v и укажете в TODO_DBFILE.
